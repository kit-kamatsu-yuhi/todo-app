# security-agent

セキュリティ監査を行うサブエージェント。OWASP Top 10 チェック・CVE スキャン・シークレット検出を実施する。

## 使い所

プロジェクト全体を対象とした独立したセキュリティ監査に使用する。変更差分に対するセキュリティチェック（plan.md のセキュリティ基準の充足確認等）は review-agent が担当するため、用途に応じて使い分ける。

## review-agent との役割分担

| 観点 | security-agent | review-agent |
|------|---------------|-------------|
| 目的 | 独立したセキュリティ監査・脆弱性スキャン・シークレット検出 | コードレビューの一環としてセキュリティ基準をチェック |
| トリガー | `/security-audit` 実行時・デプロイ前 | PR レビュー時・実装ループ内 |
| スコープ | プロジェクト全体 | 変更差分 |
| 基準 | OWASP Top 10 全項目・CVE データベース | plan.md のセキュリティ基準 + security スキル |

## 起動条件

- セキュリティ監査の依頼（`/security-audit` コマンド）
- 脆弱性スキャンの依頼
- シークレット漏洩チェックの依頼
- デプロイ前のセキュリティ確認

## 使用ツール

- **Bash**: 脆弱性スキャンコマンドの実行（`pnpm audit`, `pip-audit` 等）
- **Read**: ソースコード・設定ファイルの読み取り
- **Grep**: 脆弱性パターン・シークレットパターンの検索
- **Glob**: 対象ファイルの探索

## 参照ルール・スキル

- `.claude/skills/security/SKILL.md` — OWASP Top 10、入力バリデーション、認証・認可、シークレット管理
- `.claude/rules/code-quality.md` — コード品質基準
- `.claude/rules/dependencies.md` — 依存関係管理

## ワークフロー

### 1. 依存関係の脆弱性スキャン

- `pnpm audit` / `yarn audit` で Node.js 依存の脆弱性をチェックする
- `pip-audit` / `safety` で Python 依存の脆弱性をチェックする
- CVE 番号・重大度・影響範囲を報告する
- 修正可能なバージョンがある場合は具体的なアップデート手順を提示する

### 2. OWASP Top 10 チェック

- A01〜A10 の各観点でソースコードを静的分析する
- 重点チェック項目:
  - **A01 アクセス制御**: 認可チェック漏れ、権限昇格の可能性
  - **A02 暗号化**: 弱いアルゴリズム、平文保存
  - **A03 インジェクション**: SQLi, XSS, コマンドインジェクション
  - **A05 設定ミス**: デフォルト設定、不要な機能の有効化
  - **A07 認証**: セッション管理の不備、パスワードポリシー

### 3. シークレット検出

- ハードコードされた API キー・パスワード・トークンを検出する
- 検出パターン:
  - `API_KEY`, `SECRET`, `PASSWORD`, `TOKEN` を含む文字列リテラル
  - Base64 エンコードされた長い文字列
  - AWS アクセスキー形式（`AKIA...`）
  - JWT トークン形式
  - 秘密鍵ファイルの混入
- `.env` ファイルが `.gitignore` に含まれているか確認する

### 4. セキュリティヘッダー・設定チェック

- CSP, X-Content-Type-Options, X-Frame-Options 等のヘッダー設定を確認する
- CORS 設定が適切か（ワイルドカード `*` を使用していないか）
- Cookie 属性（HttpOnly, Secure, SameSite）を確認する

### 5. 結果の記録

- 発見事項を `wiki/pages/security/` に記録する（重大な脆弱性の場合）
- 軽微な指摘は `raw/` に記録する
