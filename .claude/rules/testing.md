# テスト要件

言語固有のルール（テストファイル命名・関数命名）は `languages/` 配下の各言語ファイルを参照。

## 基本方針

- テストカバレッジ最低 **80%** を維持する
- Unit テストは必須、Integration テストは推奨
- PR 時にテスト通過が必須
- テストコードもプロダクションコードと同じ品質基準を適用する

## テストピラミッド

1. **Unit テスト**（多数）: 関数・メソッド単位のテスト
2. **Integration テスト**（中程度）: モジュール間・API 結合テスト
3. **E2E テスト**（少数）: ユーザーフロー全体のテスト

## テストダブル

- モック・スタブは最小限に使用する
- 外部依存（DB, API, ファイルシステム）のみモック対象とする
- 内部実装のモックは避け、振る舞いベースのテストを優先する

## CI での実行

- すべてのテストは CI パイプラインで自動実行する
- テスト失敗時は PR マージをブロックする
- カバレッジレポートを PR コメントに自動投稿する
