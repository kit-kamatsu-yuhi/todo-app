# デプロイ前検証

デプロイ前に必要な検証を実施する。

## チェックリスト

1. **ルール準拠確認**
   - `testing` rule: カバレッジが基準（80%以上）を満たしているか
   - `code-quality` rule: Biome/ruff/mypy のチェックが通過しているか
   - `naming` rule: 命名規則に従っているか
   - `commit-message` rule: コミットメッセージが規約に準拠しているか
   - `dependencies` rule: lockファイルがコミットされているか、バージョンが固定されているか
   - `code-review` rule: レビュー承認を得ているか
   - `doc-management` rule: wiki / raw ドキュメントが更新されているか

2. **テスト通過確認**
   - 全テストが通過しているか確認する

3. **マイグレーション確認**
   - 未適用のマイグレーションがないか確認する
   - マイグレーションのロールバック手順が準備されているか確認する

4. **環境設定チェック**
   - 環境変数が正しく設定されているか確認する
   - シークレットがシークレットマネージャーに登録されているか確認する

5. **セキュリティ確認**
   - `security` skill の観点でチェックする
   - 依存関係の脆弱性チェック

## 出力

- 全チェック項目の結果（PASS / FAIL / WARN）
- FAIL がある場合はデプロイを推奨しない旨を明示する
