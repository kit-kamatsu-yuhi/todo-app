# コスト見積もりガイド

## 見積もり手順

1. アプリ要件からサービス構成を決定する
2. 各サービスの料金パラメータを特定する
3. `references/service-catalog.md` の料金目安を参照する
4. 月額合計を算出する

## 構成パターン別コスト目安

### パターン A: 学習・プロトタイプ（最小構成）

| サービス | 構成 | 月額目安 |
|---------|------|---------|
| App Runner | 0.25 vCPU / 0.5 GB | $5 |
| RDS | db.t3.micro | $15 |
| ECR | 1GB | $0.10 |
| Secrets Manager | 2 シークレット | $0.80 |
| **合計** | | **$21** |

App Runner はリクエストがないとスケールゼロで課金が止まるため、開発用途では実質コストが下がる。

### パターン B: 小規模プロダクション

| サービス | 構成 | 月額目安 |
|---------|------|---------|
| App Runner | 1 vCPU / 2 GB | $30 |
| RDS (Multi-AZ) | db.t3.small | $50 |
| ALB | 基本料金 + LCU | $25 |
| AWS WAF | Core ルールセット | $10 |
| CloudWatch | ログ 5GB + アラーム 5個 | $5 |
| NAT Gateway | 1台 | $35 |
| **合計** | | **$155** |

### パターン C: サーバーレス（Lambda ベース）

| サービス | 構成 | 月額目安 |
|---------|------|---------|
| Lambda | 100万リクエスト/月 | $0（無料枠内） |
| API Gateway | 100万リクエスト/月 | $3.50 |
| DynamoDB | オンデマンド、100万書込/月 | $1.25 |
| **合計** | | **$5** |

低トラフィックなら無料枠でまかなえる。

## コスト削減のポイント

- **スケールゼロ**: App Runner / Aurora Serverless / Lambda はアイドル時の課金が抑えられる
- **リザーブドインスタンス**: RDS の1年 RI で最大40%削減
- **Savings Plans**: Fargate の Compute Savings Plans で最大20%削減
- **NAT Gateway 回避**: VPC エンドポイントを使い NAT 経由のトラフィックを減らす
- **ログ保持期間**: CloudWatch Logs の保持期間を30日に設定し、長期保存は S3 に転送する
