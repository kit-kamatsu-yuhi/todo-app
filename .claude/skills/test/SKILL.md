# テスト生成・実行・カバレッジ

$ARGUMENTS に対してテストを生成・実行する。

## 手順

1. **対象コードの分析**
   - 対象ファイルの関数・メソッドを一覧化する
   - 既存テストの有無を確認する
   - テストギャップ（テストされていない箇所）を特定する

2. **テスト生成**
   - `testing` skill の方針に従う
   - 正常系・異常系・境界値のテストケースを網羅する
   - テストダブル（Mock/Stub）は外部依存にのみ使用する
   - テスト命名規約に従う

3. **テスト実行**
   - 生成したテストを実行する
   - 失敗したテストの原因を分析し、テストまたはプロダクションコードを修正する

4. **カバレッジ計測**
   - カバレッジレポートを生成する
   - ブランチカバレッジも確認する
   - 最低カバレッジ 80% を満たしているか確認する

5. **レポート出力**
   - テスト結果のサマリー（pass/fail/skip）
   - カバレッジ率（行カバレッジ、ブランチカバレッジ）
   - カバレッジが低い箇所の一覧と改善提案
